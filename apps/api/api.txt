## 🔐 AUTHENTICATION ARCHITECTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PUBLIC SITE (apps/web) auth rules:
  → Patients: email+password OR Google OAuth2
  → 2FA: OPTIONAL for patients
  → Passkey: OPTIONAL for patients
  → Session: JWT access (15min) + refresh (7 days)
  → Rate limiting: strict (DDoS protection)

ADMIN PANEL (apps/admin) auth rules:
  → ALL admin users: 2FA MANDATORY (cannot disable)
  → SSO: MANDATORY default (Google Workspace / SAML)
  → Passkey: MANDATORY for SUPER_ADMIN
  → Session: JWT access (15min) + refresh (8 hours)
  → Device tracking: MANDATORY
  → Session: max 3 concurrent per user
  → IP allowlist: configurable per role
  → Idle timeout: 30 minutes → force re-auth
  → Suspicious activity → auto-lock + alert

### GROUP A — PUBLIC AUTH (no token required)
### Base: /api/v1/auth/
══════════════════════════════════════════════════

A-01  POST   /auth/login
      Body: { email, password, deviceFingerprint,
              totpCode? }
      Returns: { accessToken, refreshToken(cookie),
                 requiresMfa, userId, roles }
      Rate: 5 attempts/15min per IP+email
      Security: bcrypt verify, account lockout after 5 fails
                log device + IP + userAgent
                detect new device → send alert email

A-02  POST   /auth/login/mfa-verify
      Body: { userId, totpCode OR backupCode,
              deviceFingerprint }
      Returns: { accessToken, refreshToken(cookie),
                 deviceTrusted }
      Rate: 3 attempts/15min per userId
      Security: invalidate on brute force

A-03  POST   /auth/login/passkey/begin
      Body: { email }
      Returns: WebAuthn PublicKeyCredentialRequestOptions
      Security: challenge expires 5 minutes

A-04  POST   /auth/login/passkey/complete
      Body: { email, credentialResponse (WebAuthn) }
      Returns: { accessToken, refreshToken(cookie) }
      Security: verify authenticatorData + signature

A-05  GET    /auth/login/google
      Redirect: → Google OAuth2 consent screen
      Scopes: email profile openid

A-06  GET    /auth/oauth2/callback/google
      Handles OAuth2 callback
      Creates account if new user
      Returns: redirect with token params

A-07  POST   /auth/register/patient
      Body: { firstName, lastName, email, phone,
              password, confirmPassword,
              dateOfBirth, gender, deviceFingerprint }
      Returns: { userId, email, message }
      Rate: 3 registrations/hour per IP
      Security: email uniqueness check
                send verification email
                log registration device

A-08  GET    /auth/verify-email/{token}
      Activates account
      Token expires: 24 hours
      Returns: { success, message }

A-09  POST   /auth/resend-verification
      Body: { email }
      Rate: 3/hour per email
      Returns: { message }

A-10  POST   /auth/forgot-password
      Body: { email }
      Returns: { message } (always 200, prevent enumeration)
      Rate: 3/hour per email
      Security: token expires 1 hour, single use

A-11  POST   /auth/reset-password/{token}
      Body: { newPassword, confirmPassword }
      Returns: { success }
      Security: invalidate all existing sessions on success
                send password change notification email

A-12  POST   /auth/refresh
      Cookie: refreshToken (HttpOnly)
      Body: { deviceFingerprint }
      Returns: { accessToken }
      Security: refresh token rotation
                invalidate old refresh token
                detect refresh token reuse → revoke all

A-13  POST   /auth/logout
      Header: Authorization Bearer
      Cookie: refreshToken
      Returns: { success }
      Security: blacklist access token in Redis
                delete refresh token

A-14  GET    /auth/check-email
      Query: ?email=
      Returns: { exists: boolean }
      Rate: 10/min per IP

A-15  POST   /auth/passkey/register/begin
      Header: Authorization Bearer [PATIENT]
      Returns: WebAuthn PublicKeyCredentialCreationOptions
      Security: challenge expires 5 min

A-16  POST   /auth/passkey/register/complete
      Header: Authorization Bearer [PATIENT]
      Body: { credentialResponse (WebAuthn) }
      Returns: { passkeyId, name, createdAt }
      Security: verify attestation

A-17  POST   /auth/2fa/setup/begin
      Header: Authorization Bearer [PATIENT]
      Returns: { qrCodeUrl, secret, backupCodes[10] }
      Security: backup codes hashed before storage

A-18  POST   /auth/2fa/setup/verify
      Header: Authorization Bearer [PATIENT]
      Body: { totpCode }
      Returns: { enabled: true, backupCodes[10] }
      Security: verify TOTP before enabling

A-19  DELETE /auth/2fa/disable
      Header: Authorization Bearer [PATIENT]
      Body: { totpCode, password }
      Returns: { success }
      Note: ADMIN/SUPER_ADMIN cannot disable 2FA

A-20  POST   /auth/token/verify
      Body: { token }
      Returns: { valid, userId, roles, expiresAt }
      Rate: 30/min per IP
      Use: frontend token validation

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

═══════════════════════════════════════════
SESSION TRACKING (both surfaces):
═══════════════════════════════════════════

On every successful login, create session record:
  sessionId: UUID
  userId: UUID
  accessTokenJti: UUID (JWT ID claim)
  refreshTokenHash: SHA-256 of refresh token
  deviceFingerprint: hash of browser/device data
  deviceName: "Chrome on Windows 11"
  deviceType: DESKTOP/MOBILE/TABLET
  browser: parsed from User-Agent
  os: parsed from User-Agent
  ipAddress: from X-Forwarded-For or RemoteAddr
  location: reverse geocoded from IP (async)
  userAgent: full string
  createdAt: timestamp
  lastActivity: timestamp (updated on heartbeat)
  isActive: boolean
  terminatedAt: nullable
  terminatedReason: nullable

Store in: PostgreSQL sessions table + Redis (fast lookup)
Redis key: session:{sessionId} TTL = refresh token lifetime

On every authenticated request:
  Validate JWT claims
  Check JWT jti not in blacklist (Redis)
  Update lastActivity (Redis async, not blocking)
  Check session is still active

═══════════════════════════════════════════
DEVICE TRACKING:
═══════════════════════════════════════════

Device fingerprint components (hashed):
  userAgent + screen resolution + timezone +
  language + platform + installed fonts (partial)

On new device detected (no matching fingerprint):
  → Send security alert email
  → Log as new_device event
  → For admin: BLOCK login until user approves device
                via email link OR approval API
  → For patient: warn but allow

TrustedDevice record:
  deviceId, userId, fingerprint (hashed)
  name, browser, os, firstSeen, lastSeen
  isTrusted, approvedAt, approvedFromIp

═══════════════════════════════════════════
PASSKEY (WebAuthn) IMPLEMENTATION:
═══════════════════════════════════════════

Library: spring-security-webauthn
         or webauthn4j-spring-security

PasskeyCredential record:
  credentialId (base64url)
  userId, publicKey (COSE)
  signCount (replay attack prevention)
  aaguid (authenticator type)
  name (user-defined, e.g. "iPhone Face ID")
  createdAt, lastUsed, deviceType

Registration flow:
  /auth/passkey/register/begin
    → generate challenge (random 32 bytes)
    → store challenge in Redis (5min TTL)
    → return PublicKeyCredentialCreationOptions

  /auth/passkey/register/complete
    → verify attestation
    → verify challenge matches
    → extract + store public key
    → store credential record

Authentication flow:
  /auth/passkey/begin
    → find user credentials by email
    → generate new challenge
    → return PublicKeyCredentialRequestOptions

  /auth/passkey/complete
    → verify assertion
    → verify challenge
    → verify signature against stored public key
    → increment signCount (detect cloned authenticators)
    → issue JWT

SSO (Admin only):
═══════════════════════════════════════════

Google Workspace SSO:
  Spring Security OAuth2 client
  Validate hd (hosted domain) claim
  Only allow configured workspace domain
  Map Google user → Admin user by email
  If no admin user exists → deny access (no auto-create)
  After SSO: STILL require MFA

SAML 2.0 (enterprise):
  Spring Security SAML2
  SP metadata: /admin/auth/saml/metadata
  ACS URL: /admin/auth/sso/saml/callback
  Sign assertions + encrypt
  Map SAML attributes → admin user attributes
  After SAML: STILL require MFA

═══════════════════════════════════════════
API TOGGLE SYSTEM (SUPER_ADMIN):
═══════════════════════════════════════════

ApiControlFilter (Spring Filter, early in chain):
  On every request:
    1. Check maintenance mode (Redis key: maintenance:mode)
       If true + IP not in allowedIps → return 503
    2. Check IP blocklist (Redis set: blocked:ips)
       If IP in set → return 403
    3. Check endpoint enabled (Redis key: api:disabled:{method}:{path})
       If exists → return 503 with reason from value
    4. Pass to next filter

ApiEndpoint entity (persisted in DB):
  id, method, path, group, description
  isEnabled (default true)
  disabledAt, disabledBy, disabledReason
  disableUntil (nullable, for temp disable)

Sync to Redis on startup:
  Load all disabled endpoints → Redis
  Check disableUntil on scheduler (every minute)
    → auto re-enable if past disableUntil

Metrics (separate):
  Redis counters: api:count:{method}:{path}
  Redis error counters: api:error:{method}:{path}
  Expires: 24h rolling window

═══════════════════════════════════════════
RATE LIMITING IMPLEMENTATION:
═══════════════════════════════════════════

Redis sliding window algorithm:
  Key: ratelimit:{identifier}:{endpoint}
  Value: sorted set of timestamps
  Window: configured per endpoint
  Limit: configured per endpoint

Rate limit identifiers:
  Public endpoints: IP address
  Authenticated: userId (more generous limits)
  Admin auth: IP + userId (strictest)

On limit exceeded:
  Return 429 Too Many Requests
  Headers:
    X-RateLimit-Limit: configured limit
    X-RateLimit-Remaining: 0
    X-RateLimit-Reset: timestamp when resets
    Retry-After: seconds to wait

Configurable per endpoint (via AC-12):
  Default public: 60 req/min
  Search/search: 20 req/min
  Auth login: 5/15min
  Booking: 10/min (authenticated patient)
  Admin auth: 3/15min
  File upload: 5/min

HIGH TRAFFIC OPTIMIZATION (public site):
═══════════════════════════════════════════

Cache layers:
  L1: Caffeine in-process (very short TTL 30s)
      for most-requested endpoints (home stats, hero)
  L2: Redis (per-endpoint TTL, see GROUP B-H)

Cache key strategy:
  Public endpoints: "cache:{endpoint}:{hash(params)}"
  Never cache patient-specific data

CDN headers on public responses:
  Cache-Control: public, max-age=60, s-maxage=300
  ETag: based on content hash
  Vary: Accept-Encoding

HTTP/2 on all endpoints

DB optimizations for public queries:
  Read replicas for all SELECT queries in public APIs
  Primary DB only for writes
  Connection pooling: HikariCP max 20 (reads: replica)
  All public APIs: @Transactional(readOnly=true)

Async operations (never block request):
  View count increments
  Search log saves
  Page view tracking
  Email sending
  Audit logging
  Notification dispatching

Security headers on ALL responses:
  X-Content-Type-Options: nosniff
  X-Frame-Options: DENY
  X-XSS-Protection: 1; mode=block
  Strict-Transport-Security: max-age=31536000
  Content-Security-Policy: (strict policy)
  Referrer-Policy: strict-origin-when-cross-origin
